* Commands To Execute :-
	- git init :- Initializes A New Git Repository In The Current Directory.
	- git init <directory> :- Creates A New Git Repository In The Specified Directory.
	- git clone https://github.com/Nilay-Kalaskar/Aspect_Flutter_Batch.git :- To Clone The GitHub Repository. This Command Is Used To Clone The Repositories Which Are Already Available In The Remote Repository.
	- Close The Git Bash Application.
	- git status :- To Check The Status. This Command Shows The Status Of The Working Tree And The Staging Area.
	- git log :- To Check The Logs Of Commits Made. This Command Will Help You To See The Logs Of All The Commits Made. Displays The Commit History Of The Current Branch.
	- git add <File_Name With Dot And It's extension> :- To Add A Single Specific File To Repository. [Files Present In The Working Area Will Move To The Staging Area].
	- git add . OR git add –all :- To Add Multiple Files To Repository. [Files Present In The Working Area Will Move To The Staging Area].
	- git config --global user.email "Email_ID With The Domain" :- Set Your Email Address.
	- git config --global user.name "First_Name_Last_Name" :- Set Your Username.
	- git commit -m "Insert Comment" :- To Commit The Staged Files. Files That Are Available In The Staging Area Will Be Committed To The Local Repository.
	- git push OR git push origin main :- Push The Changes To Repository.
	- git fetch origin main :- Pulling Without Pulling Command_1.
	- git merge origin/main :- Pulling Without Pulling Command_2.
